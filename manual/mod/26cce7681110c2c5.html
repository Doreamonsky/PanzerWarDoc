    22.6.2 Lua 中 self. 与 self: 的区别

    在 Lua 编程语言中，self. 和 self: 之间的区别主要与方法调用和表中的函数有关。它们都用于访问表（table）中的字段和方法，但是在传递参数时有所不同。

    1. self.：当你在表（table）中使用 self. 调用一个函数时，你需要显式地传递表的实例作为第一个参数。这种情况下，函数的定义通常如下：
    luaCopy code
local myTable = {} function myTable.func(self, arg1, arg2)     -- 在这里，self 是表的实例     -- arg1 和 arg2 是其他参数 end

    调用这个函数时，你需要显式地提供表的实例：

    luaCopy code
local instance = myTable instance.func(instance, arg1, arg2)
    1. self:：使用 self: 调用表中的函数时，Lua 会隐式地将表的实例作为第一个参数传递给函数。这种情况下，函数的定义通常如下：
    luaCopy code
local myTable = {} function myTable:func(arg1, arg2)     -- 在这里，self 是表的实例，已隐式地传递     -- arg1 和 arg2 是其他参数 end

    调用这个函数时，你不需要显式地提供表的实例：

    luaCopy code
local instance = myTable instance:func(arg1, arg2)

    总结：self. 和 self: 的主要区别在于传递表实例的方式。使用 self. 时，需要显式地将实例作为第一个参数传递；而使用 self: 时，表实例会被隐式地传递。在面向对象编程中，self: 通常更加方便，因为它可以简化代码。
